### Visual identity decisions (extracted from app/index.html — canonical)
- **Dark slate theme**, not light. Base `#0f1419` with a subtle radial highlight. This is a deliberate "control-room / cockpit" aesthetic fitting the pilot's-checklist metaphor in the PRD.
- **Semantic color is the core design decision, not decoration.** green `#2ecc71` = ready/fresh, amber `#f5a623` = stale/warning, red `#e74c3c` = dead/critical, blue `#4f8cff` = action. The gauge, brand dot, blocker icons, freshness pulses, and F&F clock all speak this one language. **"Never a fake green checkmark"** (PRD §4.2) is a visual rule, not just copy.
- **Radial gauge is the hero element** — 230px SVG ring, 56px/800 numeral, animates from 0 on load. The whole product is "confirm the green gauge."
- **Card-based IA** — uppercase letter-spaced muted section headers, 14px radius, soft shadow `0 6px 24px rgba(0,0,0,.35)`.
- **Type:** system stack (`Segoe UI, system-ui, -apple-system, Roboto`). Zero-dependency, no web fonts — matches the project's zero-dependency front-end convention (CLAUDE.md).
- **Motion:** restrained and meaningful — gauge fill (.6s ease), freshness pulse blink (1.6s), F&F clock ticking with color escalation. Motion signals *liveness of reconciliation*, never ornament.

### Decisions carried / flagged
- [ASSUMPTION] Light mode is NOT in v1 — the dark control-room identity is the brand. Flagged for confirm.
- [ASSUMPTION] Employee portal inherits the same token system but will need a lighter-weight, mobile-friendly layout (read-only payslip/F&F). Spine notes the delta; full portal screens deferred.
- Accessibility flag: the semantic colors must pair with non-color cues (icon + text), since the entire model leans on red/amber/green — critical for color-blind users. Captured as an Accessibility Floor requirement.
